Introduccion

URL: Es un localizador de recursos uniformes en los cuales los recursos pueden cambiar con respecto al tiempo. Son muy utiles al momento de declarar hiperenlaces en una pagina web.

MIME: Multipurpose Internet Mail Extensions, son un convenciones y especificaciones para mandar imagenes, texto, audio, etc. a traves de internet, se trata de que todas las maquinas puedar mostrar cualquier alfabeto, formato de imagen, etc.

Codigos de estado HTTP: 1xx: Respuestas informativas, 3xx: Redirecciones, 4xx Errores del cliente, 5xx Errores de servidor.
Para mas informacion acerca de los codigos fuente: Wikipedia

XML: Especificación para diseñar lenguajes de marcado, que permite definir etiquetas personalizadas para descripción y organización de datos.

XHTML: EXtensible HyperText Markup Language es casi idéntico a HTML pero más estricto, esto sirve ya que hay veces que un navegador móvil no puede corregir los errores mientras que uno de escritorio si, y en XHTML al ser más estricto nos encontramos con menos errores.

HTML5: Es la quinta versión de HTML, sus principales diferencias con sus predecesores es que la estructuración de la página se hace mucho más simple, ya que al dejar los estilos de las etiquetas a CSS el código HTML se vuelve mucho mas compacto.

JavaScript: Se utiliza para dar funcionalidad a las páginas web, este contenido no se ve en la página, pero todos los procesos que esta realiza se hacen mediante JS.

PHP: Hipertext procesor es un lenguaje de programación para que tiene el fin de ayudar al servidor, se utiliza para el desarrollo web y se puede incorporar directamente en el HTML.

JSP: se utiliza para hacer páginas web dinámicas, es muy parecido a PHP pero usa Java como lenguaje.

AJAX: AJAX permite que las páginas web se actualicen de forma asíncrona mediante el intercambio de datos con un servidor web detrás de las escenas. Esto significa que es posible actualizar partes de una página web, sin volver a cargar toda la página.

¿Cuál es la diferencia entre Internet y la World Wide Web? Internet es la red de redes y WWW es el sistema que se utiliza para acceder a internet

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
El verbo POST se usa con más frecuencia para ** crear ** recursos nuevos. En particular, se usa para crear recursos subordinados. Es decir, subordinado a algún otro recurso (por ejemplo, padre). En otras palabras, al crear un recurso nuevo, POST al padre y el servicio se encarga de asociar el nuevo recurso con el padre, asignar una ID (URI de recurso nuevo), etc.
El método HTTP GET se usa para ** leer ** (o recuperar) una representación de un recurso. En la ruta "feliz" (o sin errores), GET devuelve una representación en XML o JSON y un código de respuesta HTTP de 200 (OK). En un caso de error, con mayor frecuencia devuelve un 404 (NO ENCONTRADO) o 400 (PETICIÓN MALA).
La mayoría de las veces PUT se utiliza para las capacidades de ** actualización **, PUT-ing a un URI de recursos conocido con el cuerpo de la solicitud que contiene la representación recién actualizada del recurso original.
PATCH se usa para ** modificar ** capacidades. La solicitud PATCH solo necesita contener los cambios en el recurso, no el recurso completo.
DELETE es bastante fácil de entender. Se usa para ** eliminar ** un recurso identificado por un URI.

Ciclo de vida de sistemas de informacion


Todos los sistemas de información tienen un ciclo de vida. Este es el que da un orden a lo que se tiene que hacer mientras se desarrolle. Primero que nada este debe planearse junto con el cliente para saber que se debe hacer; el segundo paso es el análisis, donde el desarrollador analiza que es lo que quiere el cliente y le muestra una propuesta, si todo sale bien podemos seguir al siguiente paso, mientras que si algo sale mal en cualquier parte del ciclo siempre debemos retroceder; Lo siguiente seria el diseño, en donde el desarrollador plasma sus ideas sobre el programa que platico y analizo con el cliente y le muestra un bosquejo, si el cliente está de acuerdo seguimos; Continuamos con la implementación, donde empezamos a codificar el programa; Una vez que está listo se hacen las pruebas necesarias y finalmente después de entregarse el producto hay un periodo de mantenimiento.
